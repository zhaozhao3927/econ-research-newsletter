# Behavioral & Experimental Economics Weekly вЂ” Design Spec

**Date:** 2026-05-26  
**Status:** Approved

---

## Overview

A weekly static-HTML newsletter tracking new papers across top economics and
psychology journals, with a focus on behavioral and experimental economics.
Automated Python pipeline fetches, filters, summarizes via Claude API, renders
to HTML, and publishes to GitHub Pages every Monday.

## Feed Sources

Monitored via RSS / API, 7-day rolling window:

**Top general economics:**
- American Economic Review (AER)
- Quarterly Journal of Economics (QJE)
- Journal of Political Economy (JPE)
- Review of Economic Studies (REStud)
- Econometrica

**Applied & policy:**
- AEJ: Applied, Micro, Macro, Policy
- Review of Economics and Statistics (REStat)
- Journal of the European Economic Association (JEEA)
- Economic Journal (EJ)

**Behavioral & experimental specialist:**
- Experimental Economics
- Journal of Economic Behavior & Organization (JEBO)
- Games and Economic Behavior (GEB)

**Working papers:**
- NBER (`nber.org/rss/new_working_papers.xml`)
- SSRN (behavioral econ network RSS)
- CEPR

**Interdisciplinary:**
- Psychological Science
- Nature Human Behaviour
- Journal of Experimental Psychology: General

All sources configurable in `config.yaml`. Add/remove without code changes.

## Filtering Logic

Two-stage, runs per paper:

1. **Keyword pre-filter** вЂ” matches against `keyword-clusters.md` (from
   bacon-research skill). Sets `isBehavioral: true/false`. Fast, zero API cost.

2. **Claude API review** вЂ” for borderline cases (1вЂ“2 keyword matches),
   Claude reads the abstract and makes the final call. Papers with 0 matches
   are marked `isBehavioral: false` automatically; papers with 3+ matches are
   marked `true` automatically. Only the 1вЂ“2 match band calls the API.

All papers are stored in the JSON regardless of `isBehavioral` value.
Filtering is UI-only вЂ” the flag drives the client-side toggle.

## HTML & UI Design

Visual style mirrors the reference repo (`follow-builders-newsletter`):
- Playfair Display masthead, IBM Plex Sans body, warm paper (`#f6f3ed`) background
- Accent color: `#c0392b` (red)

**Issue page sections:**
1. **Masthead** вЂ” newsletter name, issue number, week date range
2. **Editor's note** вЂ” auto-generated by Claude, 2вЂ“3 sentences on week's themes
3. **Filter bar** вЂ” `All Papers` (default) | `Behavioral & Experimental Only` toggle + journal chips (AER, QJE, NBERвЂ¦)
4. **Paper cards** вЂ” one per paper:
   - Title, authors, journal badge, publication date
   - Four summary fields: RQ / Method / Key Finding / Why It Matters
   - DOI link
5. **Footer** вЂ” source attribution, issue number

**Filter behaviour:**
- Client-side JS only, no page reload
- Filter state stored in URL hash (`#behavioral`, `#aer`, etc.) for shareable links
- Default: all papers visible

**Index page:**
- Cover with newsletter title and description
- Archive list: issue date, week range, paper count
- Link to each issue HTML

## Pipeline Detail

### fetch.py
- Uses `feedparser` to poll each RSS feed
- Collects entries published in the past 7 days
- Deduplicates by DOI (falls back to title+authors hash)
- Outputs list of raw paper dicts

### summarize.py
- Checks `cache/summaries.json` by DOI вЂ” skips already-summarized papers
- Calls Claude API (`claude-sonnet-4-6`) with paper title + abstract
- Prompt produces four fields: `rq`, `method`, `keyFinding`, `whyItMatters`
- Also generates `isBehavioral` flag and `tags` list
- Also generates weekly `intro.text` from all paper titles/findings
- Writes results back to cache

### build_issue.py
- Merges fetched papers with summaries
- Assigns issue number (auto-incremented from last issue in `data/issues/`)
- Writes `data/issues/becon-weekly-YYYY-MM-DD.json`

### render.py
- Reads issue JSON, renders via Jinja2 template (`templates/issue.html.j2`)
- Writes to `issues/becon-weekly-YYYY-MM-DD.html`

### update_index.py
- Scans all JSONs in `data/issues/`
- Rebuilds `index.html` from `templates/index.html.j2`
- Preserves existing entries, adds new issue at top

### run_weekly.ps1
- Orchestrates the full chain
- Sets `ANTHROPIC_API_KEY` from Windows environment
- Logs to `logs/YYYY-MM-DD.log`
- On failure: logs error, exits without pushing

### publish.ps1
- `git add data/ issues/ index.html`
- `git commit -m "Weekly issue: YYYY-MM-DD (N papers)"`
- `git push origin main`
- GitHub Pages auto-refreshes

## Error Handling

- Feed fetch failure: log and skip that source, continue with others
- Claude API failure: log, skip paper summary, include paper with empty summary fields
- Git push failure: log error, leave changes staged for manual push
- No new papers found: skip issue generation, log "no new papers this week"

- Email delivery
- Search across past issues
- Comments or interaction
